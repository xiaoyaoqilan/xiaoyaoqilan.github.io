RAG & 自动化

我做了一个 Hermes 自动链接 IMA 上传文档并整理为文件夹的 RAG 系统

2026-06-01 · QILAN

每天面对各种财经资讯、研报和公告，最烦人的动作就是：下载 PDF -> 打开浏览器 -> 找到对应的文件夹 -> 上传 -> 还要人工重命名。这种机械式无脑操作简直是在浪费生命。既然我们是数字极客，就应该用技术把这一切给自动化了。

我写了一个名为 Hermes 的自动化工具。它可以自动监控数据源、通过 CDP/Playwright 链接到 IMA 平台，并自动把下载好的文档分门别类整理成文件夹形式，无缝上传。之后，再调用后台 RAG（检索增强生成）引擎，对多源财经数据进行深度提炼。整个过程我完全不需要碰鼠标。

“把时间花在深度思考和投资决策上，而不是沦为搬运文件的表哥表姐。”

系统核心设计思路

  • 多财经渠道抓取 (Data Collector)：利用 Python 异步脚本自动订阅、监控主流的财经研报和公告源，一旦有新内容自动落库本地暂存区。
  • 智能分类与重命名 (Smart Classifier)：在上传前，利用轻量级大模型快速提取文档标题、机构、时间与核心主题，将其转换成标准文件命名格式，并计算其所属的板块文件夹路径。
  • 自动链接 IMA 上传 (Auto Uploader)：通过 Playwright 接管已登录的 Edge 浏览器会话，静默完成 IMA 对应的目录定位、新建文件夹与多线程文件上传工作，全程零人工干预。
  • 多源数据 RAG 查询 (Hybrid RAG)：构建基于 Pinecone / Milvus 的混合检索，将 IMA 的结构化文件库和实时抓取的财经快讯进行融合。即使在遇到断网或部分失效时也能通过本地缓存继续运行，尽显韧性。

极客的底色：偶尔犯傻，但绝不妥协

刚开始调试这个工具时，我因为没有给上传重试加上随机指数退避（Exponential Backoff），结果第一天直接把 IMA 端口请求给打爆了，IP 差点被封，简直像个憨憨。但好在我快速迭代了代理轮换与 CDP 保持策略。现在系统稳定地像头怪兽，每天自动运转 24 小时，替我节省了大量的案头检索与分类时间。
