把个人主页迁移到 GitHub Pages 的完整清单

2026-05-12 · 起岚

个人主页如果只有一句欢迎语和几个跳转链接，很难获得搜索流量。迁移到 GitHub Pages 时，我更希望它变成一个稳定的内容中心。

上线前要检查什么

  • 首页标题要直接说明你是谁、写什么，而不是只写“主页”。
  • 每篇文章都要有独立 URL、标题、摘要、发布时间和内链。
  • 准备 sitemap.xml、robots.txt 和 RSS，方便搜索引擎与读者订阅。
  • 外链入口要少而清楚，优先放你真的持续维护的平台。

GitHub Pages 的好处是部署简单、版本清晰、可回滚。只要内容结构合理，它完全可以承载一个轻量但专业的独立博客。
